# Axiom Memory System v1.0

> "Memory is not storage. Memory is understanding."

# Purpose

The Memory System gives Axiom persistent knowledge across time.

Its purpose is to remember:

• Projects

• Conversations

• Documents

• Research

• Relationships

• Skills

• Preferences

• Discoveries

Memory should continuously become more valuable.

I'd add one concept that I think could become Axiom's superpower
Memory Layers

Instead of one database, imagine overlapping layers:

───────────────────────────────
Identity Layer
───────────────────────────────
Long-Term Goals
───────────────────────────────
Projects
───────────────────────────────
Research
───────────────────────────────
Skills
───────────────────────────────
Conversations
───────────────────────────────
Working Memory
───────────────────────────────

When you're asking about a Counter-Strike AI project, Axiom doesn't need to search your aviation notes first. It starts with the project layer, then expands outward only if needed. That makes retrieval both faster and more relevant.

The idea I'm most excited about
Memory should grow, not just accumulate.

Imagine you've spent six months learning computational geometry.

Instead of simply storing hundreds of notes, Axiom notices:

You frequently use convex hulls with point clouds.
Those ideas also appear in computer vision.
They connect to SLAM, drone navigation, and CAD.
You're now exploring robotics.

One day it says:

"You've independently studied four subjects that all rely on spatial reasoning. Would you like me to create a unified learning map showing how they connect?"

That's not just memory.

That's a second brain that helps organize and deepen your understanding rather than acting as a passive archive.
